## v2.2 第五次技術會議變更（2026-06-29，凍結前最後確認）

| # | Section | 變更內容 | 來源 |
|---|---------|---------|------|
| 1 | 資料模型 identification | 移除自然人 `identification.type` 的 `national_id`、`passport`（證期局草案未納入自然人證件號碼）；自然人改以 `account_id` 辨識，`identification`（type/number）僅法人大額交易適用 | MaiCoin 提案 |
| 2 | 資料模型 date_of_birth | 由 v2.1「至少提供年份」收緊為**必須完整出生年月日 `YYYY-MM-DD`**（證期局草案僅接受完整日期）；regex 改 `^\d{4}-\d{2}-\d{2}$` | MaiCoin 提案 |
| 3 | 資料模型 name / 居留證 | `name` 來源依證件類型認定（身分證→中文、護照→羅馬拼音、居留證→證上名）；因自然人不交換證件號碼，不另設居留證 identification type | XREX 提問、幣託 5/27 提出 |
| 4 | 資料模型 account_id / address | 釐清：兩者 JSON key 一律必存（不設 Optional），值取不到時以空字串 `""` 表示、接收方視為「未提供」；自律規範對是否擇一由法遵持續釐清，key 結構不變 | 會議決議（主席） |
| 5 | POST /transfer 確認逾時放行（發送方） | 新增「確認逾時的鏈上放行條件（發送方）」：因證期局不接受補件，原則須 `accepted` 才上鏈；受益方逾時未回之例外——發送方能以可辨識方式獨立確認受益方 VASP 身分與地址歸屬並留存紀錄者，得放行；無法辨識或 `rejected` 則不得放行。為「廣播優先原則」於境內無補件機制下之收斂 | 何亞 Andy、跨鏈提問 |

> **已於 PR #8 先前 commit 落地、本次確認不再變更**：confirm `status` 僅 `accepted`/`rejected`、broadcast-first 接收方 grace window、`amount_twd` 門檻基準、CAIP-2 / CAIP-19 鏈與資產識別。
>
> **測試但規格保留**：資產先上鏈後補件機制——證期局確定不接受補件，故點對點測試階段不測；惟相關 `expires_at` / grace window 機制於規格保留，未來如有彈性可啟用。

**v2.1.1 為 breaking change** — 移除 `company_registration` 類型、`encryption` 改必填。由於 v2.1 規格尚未有任何正式實作，不需向後相容過渡期。所有實作方請直接使用 `business_registration` 並提供 `encryption.kid`。
</Warning>
