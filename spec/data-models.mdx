## Originator（發起人：自然人/法人）

```json
originator:
{
  "type": "natural_person | legal_person",
  "name": "string",
  "account_id": "string (at least one of account_id or address required)",
  "address": "string (blockchain address, at least one of account_id or address required)",
  "memo": "string (Memo/Tag, if applicable)",
  "identification": {
    "type": "lei | tax_id | business_registration (法人專用；自然人不帶 type/number)",
    "number": "string (法人統編 / LEI / 登記字號)",
    "country": "string (ISO 3166-1 alpha-2；自然人於大額交易帶國籍)"
  },
  "date_of_birth": "string (YYYY-MM-DD, 完整出生年月日)",
  "place_of_birth": "string",
  "physical_address": {
    "country": "string (ISO 3166-1 alpha-2, e.g. TW)",
    "city": "string (city name)"
  }
}
```

## Beneficiary（受益人：自然人/法人）

```json
beneficiary:
{
  "type": "natural_person | legal_person",
  "name": "string",
  "account_id": "string (at least one of account_id or address required)",
  "address": "string (blockchain address, at least one of account_id or address required)",
  "memo": "string (Memo/Tag, if applicable)",
  "identification": {
    "type": "lei | tax_id | business_registration (法人專用；自然人不帶 type/number)",
    "number": "string (法人統編 / LEI / 登記字號)",
    "country": "string (ISO 3166-1 alpha-2；自然人於大額交易帶國籍)"
  },
  "date_of_birth": "string (YYYY-MM-DD, 完整出生年月日)",
  "place_of_birth": "string",
  "physical_address": {
    "country": "string (ISO 3166-1 alpha-2, e.g. TW)",
    "city": "string (city name)"
  }
}
```

### Identification Types

`identification`（type/number）**僅法人適用**，且僅在大額交易（等值 TWD ≥ 30,000）時必填。自然人不帶 `identification.type` / `number`。

| Type | 適用主體 | 說明 | 法人優先順序 |
|------|---------|------|--------------|
| lei | 法人 | **[v2.1 新增]** Legal Entity Identifier（法人識別碼） | 1（最優先） |
| tax_id | 法人 | **[v2.1 新增]** 統一編號 | 2 |
| business_registration | 法人 | **[v2.1 新增]** 公司登記字號 | 3 |

<Note>
**法人 identification type 優先順序**：`lei` > `tax_id` > `business_registration`。若有 LEI 碼應優先使用。
</Note>

<Warning>
**[v2.2 第五次技術會議] 移除自然人證件號碼（MaiCoin 提案）**

依證期局提供之反洗錢辦法草案，自然人辨識**未納入**身分證字號及護照號碼，故移除 `identification.type` 的 `national_id`、`passport` 兩種類型。自然人改以業者配發之 **`account_id`（帳戶編碼）** 辨識；大額交易（≥ 30,000 TWD）另帶 `identification.country`（國籍）、`date_of_birth`、`place_of_birth`、`physical_address`，但**不帶 `type` / `number`**。

**`name` 欄位的來源依用戶證件類型認定**：身分證 → 中文姓名；護照 → 羅馬拼音；居留證 → 居留證上之名字。居留證僅作為 `name` 之來源依據，因自然人不交換證件號碼，故**不另設居留證 identification type**。
</Warning>

<Warning>
**[v2.1 Breaking Change]** 移除舊有的 `company_registration` 類型。v2.1 起法人登記字號一律使用 `business_registration`。由於 v2.1 規格尚未有任何實作，此變更不需過渡期。
</Warning>

### date_of_birth 格式 [v2.2 第五次技術會議變更]

`date_of_birth` 須為**完整出生年月日** `YYYY-MM-DD`：

| 格式 | 範例 | 說明 |
|------|------|------|
| `YYYY-MM-DD` | `"1990-05-15"` | **必填格式（完整出生年月日）** |

驗證用正規表達式：`^\d{4}-\d{2}-\d{2}$`

<Warning>
**[v2.2 第五次技術會議] 改為完整出生年月日（MaiCoin 提案）**

依證期局提供之反洗錢辦法草案，僅接受完整出生日期。原 v2.1「至少提供年份」（接受 `YYYY` / `YYYY-MM`）之放寬規則**作廢**，一律須帶 `YYYY-MM-DD`。
</Warning>

### physical_address 結構 [v2.1 變更]

`physical_address` 由原本的單一字串改為結構化物件（與其他 PII 欄位一同透過外層 `private_info` envelope 加密傳輸）：

| 欄位 | 類型 | 說明 |
|------|------|------|
| country | string | ISO 3166-1 alpha-2 國家碼（如 `"TW"`） |
| city | string | 城鎮名稱 |

<Warning>
**[v2.2 釐清] 大額交易必填欄位（提案 9）**

依《防制洗錢及打擊資恐注意事項自律規範》第十二條之一，移轉虛擬資產等值新臺幣三萬元以上者，應包含接收人之居住、出生或註冊營業之國家及城鎮名稱。對應欄位為 `physical_address`（v2.1 已加入結構化 `{country, city}`），必填規則如下：

| 交易金額（等值 TWD） | `originator.physical_address` | `beneficiary.physical_address` |
|------|------|------|
| ≥ 30,000 | **必填** | **必填** |
| < 30,000 | optional | optional |

**門檻判斷基準 [v2.2 釐清]**：本門檻以 **`transaction.amount_twd`（等值新台幣）** 為準，由發送方依交易當下匯率認定；不以 `amount_usd` 判斷，避免各家美元匯率不一導致同筆交易門檻結果不一致。

`city` 欄位的標準化用詞（如 `"New Taipei City"` vs `"新北市"`）待確認，將參考 ISO 3166-2:TW 或其他標準。
</Warning>

### account_id / address 條件式必填 [v2.1 新增]

`account_id`（帳戶編碼）與 `address`（區塊鏈地址）**至少須提供其一**：

| 情境 | account_id | address | 說明 |
|------|-----------|---------|------|
| 用戶有獨立鏈上地址 | 可選 | 必填 | 一般情況 |
| 從水庫地址（omnibus address）發送 | 必填 | 可選 | 帳戶編碼與用戶 1:1 對應 |
| 兩者皆有 | 必填 | 必填 | 最佳實務 |

<Note>
**背景**：交易所在處理客戶虛擬資產發送時，可能從水庫地址（omnibus/pool address）發送，而非分配給發起人獨一無二的區塊鏈地址。經公會秘書處與主管機關確認，自律規範允許以帳戶編碼替代區塊鏈地址，因帳戶編碼與用戶為 1:1 對應。此做法亦與國際慣例一致（歐盟 TFR、新加坡 PSN02 皆採 account number or blockchain address）。
</Note>

<Warning>
**[v2.2 第五次技術會議] 欄位 key 恆存、空值以空字串表示**

為符合自律規範「須有此資訊」之文字、並保持 schema 形狀穩定與 transaction log 完整，`account_id` 與 `address` 之 **JSON key 一律必須存在**（不省略、不設為 Optional）。若某值依上表情境確實取不到（例：水庫地址無對應 `account_id`、或受益方端發起時尚未取得對方帳戶編碼），**該值以空字串 `""` 表示**；接收方須將空字串視為「未提供」，改抓另一個可用欄位判斷。

> 自律規範對「錢包資訊」之確切定義（`account_id` 與 `address` 是否須兩者皆有值，或可擇一）由法遵與證期局持續釐清；屆時僅調整「value 何時允許為空」，**欄位 key 結構不變**。
</Warning>

## Transaction

```json
{
  "tx_hash": "string",
  "vout": "integer (UTXO output index, UTXO 鏈才需要)",
  "network": "string (鏈短名別名, e.g. ethereum；過渡期保留)",
  "chain_id": "string (CAIP-2 chain_id, e.g. eip155:1)",
  "asset_id": "string (CAIP-19 asset_id, e.g. eip155:1/erc20:0xdac17...)",
  "amount": "string (decimal, 原幣別數量)",
  "amount_twd": "string (decimal, 等值新台幣；自律規範門檻判斷依據)",
  "amount_usd": "string (decimal, 等值美元；純國際參考)",
  "memo": "string (Memo/Tag, if applicable)",
  "block_number": "integer",
  "originated_at": "string (ISO 8601)",
  "confirmed_at": "string (ISO 8601)"
}
```

<Note>
**[v2.2 新增] `vout` 欄位（提案 13）**

`vout` 為 UTXO output index。BTC 等 UTXO 鏈一筆交易可能含多個 output，僅 `tx_hash` 不足以定位受益人收到的 output，故新增此欄位。僅在 CAIP-2 `chain_id` 對應 UTXO 鏈（如 Bitcoin `bip122:000000000019d6689c085ae165831e93`）時提供；非 UTXO 鏈可省略或為 `null`。
</Note>

## 支援的資產類型 [v2.2：改採 CAIP-19]

<Note>
**[v2.2 變更] 資產識別改採 CAIP-19 標準**

資產以 [CAIP-19](https://chainagnostic.org/CAIPs/caip-19) `asset_id` 為**正規識別碼**（填入 API payload 的 `asset_id`）。**凡可由 CAIP-19 表示、且位於上述支援 CAIP-2 鏈上的資產皆受支援，不再逐欄列舉。** 下表僅列常見資產的標準範例。

**wrapped / bridged 資產**：同一 symbol 於非原生鏈上多為 wrapped／bridged 代幣（如 BTC on `bsc` 實為 BTCB）。CAIP-19 `asset_id` 已內含合約地址，能精確區分，請以 `asset_id` 為準。
</Note>

## 需要 Memo/Tag 的網路 [v2.0 新增]

以下網路在進行地址查詢與交易時，可能需要同時提供 Memo 或 Destination Tag：

| `chain_id`（CAIP-2） | Memo 欄位名稱 | 說明 |
|------|------|------|
| `xrpl:0` | Destination Tag | 數字型態，用於區分同一地址下的不同用戶 |
| `stellar:pubnet` | Memo | 文字或數字型態 |
| `cosmos:cosmoshub-4` | Memo | 文字型態 |
| `antelope:aca376f206b8fc25a6ed44dbdc66547c` | Memo | 文字型態 |

<Warning>
發起方 VASP 在查詢地址及發送 Transfer 時，若目標網路屬於上述類型，**必須**提供對應的 memo 欄位。
</Warning>
